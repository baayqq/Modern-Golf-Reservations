Ringkasan Aplikasi

Modern Golf Reservations adalah aplikasi berbasis web untuk mengelola reservasi tee time dan pembayaran di lapangan golf. Aplikasi dirancang sederhana, responsif, dan mudah digunakan untuk kebutuhan operasional harian.

  • Antarmuka: Dashboard, Tee Time Reservation (Create & Manage), Manage POS, dan Profile.
  • Data disimpan lokal melalui SQLite (web menggunakan sqlite3.wasm).
  • Form reservasi mendukung hingga 4 pemain dengan ketua sewa sebagai pemain pertama.
  • Tampilan daftar pemain ditunjukkan di Dashboard dengan tabel bertepi kotak rapi.

Fitur Utama

1) Login

  • Autentikasi sederhana untuk mengakses fitur reservasi dan POS.
  • Menyimpan sesi pengguna agar tetap masuk hingga logout.

2) Tee Time Reservation — Create

  • Membuat reservasi dengan tanggal, jam mulai, jumlah pemain (dropdown 1–4), dan nama pemain 1–4.
  • Validasi otomatis: jumlah pemain maksimal 4; nama pemain 2–4 wajib diisi bila jumlah ≥ 2/3/4.
  • Catatan opsional untuk detail tambahan.

3) Tee Time Reservation — Manage Reservation

  • Meninjau, mengedit, atau membatalkan reservasi yang sudah dibuat.
  • Gerbang pembayaran: dari sini, Anda dapat melanjutkan ke POS untuk membuat invoice dan mencatat pembayaran.

4) Dashboard

  • Melihat daftar reservasi terbaru dengan tabel bergaris (boxed grid).
  • Menampilkan Nama Pemain 1–4 per reservasi.

5) Manage POS (Invoice)

  • Membuat invoice terkait reservasi dan mencatat pembayaran (diakses dari Manage Reservation).
  • Biaya mengacu pada konfigurasi di lib/config/fees.dart (mis. green fee, cart, caddie, dll).

Alur Penggunaan (Step by Step)

A. Login

  1. Buka aplikasi web Modern Golf Reservations.
  2. Masukkan kredensial (username dan password) pada halaman Login.
  3. Klik Login. Jika berhasil, Anda akan dibawa ke Dashboard.

B. Membuat Pesanan (Create Tee Time)

  1. Buka menu Tee Time Reservation kemudian pilih Create.
  2. Pilih Tanggal menggunakan pemilih tanggal.
  3. Pilih Jam Mulai menggunakan pemilih waktu.
  4. Pilih Jumlah Pemain dari dropdown (1–4). Jumlah ini menentukan wajib/tidaknya kolom nama pemain 2–4.
  5. Isi Nama Pemain 1 (Ketua Sewa). Wajib diisi.
  6. Jika jumlah pemain ≥ 2, isi Nama Pemain 2. Demikian pula untuk pemain 3 dan 4 bila jumlah ≥ 3/4.
  7. Isi Catatan bila diperlukan (opsional).
  8. Klik Save. Jika valid, reservasi tersimpan dan aplikasi akan kembali ke kalender booking dengan tanggal yang dipilih.
Catatan: Jika ada kesalahan (mis. jumlah pemain di luar 1–4 atau nama pemain kurang), sistem menampilkan pesan validasi yang jelas.

C. Kelola Reservasi (Manage Reservation)

  1. Buka menu Tee Time Reservation lalu pilih Manage (kelola reservasi).
  2. Cari reservasi yang ingin diproses. Anda bisa meninjau detail, mengubah, atau membatalkan jika diperlukan.
  3. Untuk melakukan pembayaran, pilih aksi Proses Pembayaran / Buat Invoice. Sistem akan mengarahkan Anda ke POS/Invoice.

D. Pembayaran melalui POS/Invoice (via Manage Reservation)

  1. Dari halaman Manage Reservation, Anda akan diarahkan ke Manage POS untuk reservasi yang dipilih.
  2. Buat invoice dengan menambahkan item biaya sesuai konfigurasi fees.dart (green fee, cart, caddie, dll).
  3. Setel qty mengikuti jumlah pemain pada reservasi.
  4. Simpan invoice, lalu pilih Bayar untuk mencatat pembayaran. Pilih metode pembayaran yang tersedia (tunai, transfer, dsb sesuai kebutuhan aplikasi).
  5. Setelah pembayaran dicatat, status invoice berubah menjadi Lunas. Simpan atau cetak invoice bila diperlukan.

Detail Form Tee Time

  • Tanggal: format dd/mm/yyyy, dipilih via DatePicker.
  • Jam Mulai: format HH:mm (24 jam), dipilih via TimePicker.
  • Jumlah Pemain: dropdown 1–4.
  • Nama Pemain 1 (Ketua Sewa): wajib diisi.
  • Nama Pemain 2–4: wajib bila jumlah pemain ≥ 2/3/4.
  • Catatan: opsional.

Validasi dilakukan sebelum simpan: jumlah maksimal 4, serta nama pemain yang diperlukan harus terisi.

Dashboard & Daftar Pemain

Dashboard menampilkan daftar reservasi dengan tampilan tabel bertepi kotak rapi. Kolom nama pemain 1–4 akan terisi sesuai data di reservasi. Tampilan mendukung scroll untuk data banyak dan tetap responsif.

Manage Reservation

Halaman ini adalah pusat pengelolaan reservasi yang sudah dibuat. Gunakan untuk:

  • Review — melihat detail reservasi (tanggal, jam, jumlah pemain, nama pemain).
  • Edit — memperbarui data bila terjadi perubahan.
  • Cancel — membatalkan reservasi bila perlu.
  • Proses Pembayaran — mengarahkan ke Manage POS untuk membuat invoice dan menyelesaikan pembayaran.

Dengan alur ini, semua transaksi pembayaran dimulai dari Manage Reservation agar terhubung dengan reservasi yang valid.

POS/Invoice

Aplikasi menyediakan halaman Manage POS untuk membuat dan mengelola invoice terkait reservasi. Aksesnya berasal dari Manage Reservation untuk menjamin konsistensi data.

  • Biaya standar didefinisikan di lib/config/fees.dart.
  • Gunakan jumlah pemain sebagai acuan kuantitas item biaya.
  • Status invoice berubah sesuai aksi pembayaran (mis. Draft → Lunas).

Tips & Troubleshooting

  • Validasi gagal: Periksa kembali jumlah pemain dan kelengkapan nama pemain sesuai jumlah.
  • Nama pemain tidak muncul di Dashboard: Pastikan simpan berhasil dan refresh halaman Dashboard.
  • Mulai pembayaran: Pastikan Anda memulainya dari Manage Reservation agar reservasi terhubung ke invoice.
  • Browser mobile: Gunakan mode portrait untuk kenyamanan input; tabel akan otomatis bisa di-scroll.
  • Cetak invoice: Gunakan fitur cetak browser dari halaman invoice.

FAQ

Kenapa maksimal 4 pemain? Karena setiap slot tee time umumnya dibatasi 4 orang untuk menjaga alur permainan.

Bisakah menambah lebih dari 4? Tidak. Sistem membatasi input pada dropdown jumlah pemain 1–4.

Bagaimana jika saya hanya punya 2 pemain? Pilih jumlah 2, isi nama pemain 1 dan 2. Kolom 3–4 boleh dibiarkan kosong.

Catatan Teknis

  • Arsitektur: UI di folder lib/screens/, model di lib/models/, layanan database di lib/services/.
  • Model Tee Time: menyimpan playerName (ketua sewa) serta player2Name, player3Name, player4Name, dan playerCount.
  • SQLite: tabel tee_times berisi kolom untuk nama pemain 1–4 dan jumlah pemain.
